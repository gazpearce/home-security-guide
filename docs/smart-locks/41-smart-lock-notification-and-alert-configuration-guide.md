## Specifications and Comparison

| Notification Type | Default Setting | Recommended Setting | Quiet Hours Override | Critical Alert |
| ---|---|---|---|--- |
| Lock/unlock (family) | On | Off (except night hours) | No | No |
| Lock/unlock (guest) | On | On | No | No |
| Failed code attempt | On | On (5+ attempts) | Yes | Yes |
| Tamper alarm | On | On | Yes | Yes |
| Low battery | On | On (20% + 10%) | Yes | Yes (10% only) |

## FAQ

<dl>

<dt>How do I prevent notification overload?</dt>
<dd>Filter notifications by user type - only alert for guests and temporary codes. Enable Quiet Hours at night. Use per-event type control.</dd>

<dt>Can I get notifications when someone tries the wrong code?</dt>
<dd>Yes - enable failed attempt notifications. Set threshold to 3-5 attempts. Lock enters 30-second lockout after failed attempts.</dd>

<dt>Will notifications work without internet?</dt>
<dd>Push notifications require internet. If internet is down, the lock still operates locally. Check the activity log in the app when internet restores. For more information see our <a href="../smart-locks/40-smart-lock-key-fob-and-rfid-tag-guide.md">Smart Lock Key Fob and RFID Tag Guide</a>.</dd>

</dl>
